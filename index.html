  1. The Upgrade of Computer Code’s Form (inside The Editor)

    Programming languages can accomplish more than a few decades ago, but not by much, and they haven't been upgraded in a way that utilizes the new capabilities built into the computers. For example, producing a competing web browser engine as large as Chromium or WebKit can't be done in a short period of time, and the goal should be to change that. But for that kind of upgrade to happen, changes like the above image have to occur, because code has to look more like the software it actually produces, not just raw monospace type. Typical computer science mindsets won't suffice. The initial subjects of concern turn out to be different, and the areas of expertise are often outside of typical discussions.

    As explained in articles below, the obstacle to programming's progress is the confining, primitive offerings of the file format plain text, that it has been stretched beyond its bare capabilities to serve increasingly large and complex modern needs compared to the 1970s. Computer code needs to be editable in multiple newspaper-like columns, for one, to make efficient use of screen space. Plain text is too raw of a file format for programming software in the 21st century, and it doesn't allow inclusion of modern gui controls. An array of values is still not editable by interacting with an inserted data table GUI control.
